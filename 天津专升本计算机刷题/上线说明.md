# 天津专升本计算机刷题网站上线方案

当前项目是纯静态网站，不需要数据库和后端服务。上线时只要上传这些文件即可：

- `index.html`
- `styles.css`
- `app.js`
- `questions-data.js`

用户的答题记录、错题本、每日总结会保存在浏览器本地。换浏览器或换设备后记录不会自动同步。

## 长期正式推荐：腾讯云 CloudBase 静态托管

适合你长期自用，国内访问更稳，后面也方便扩展登录、数据库和云函数。

操作步骤：

1. 打开腾讯云控制台
2. 搜索 `云开发 CloudBase`
3. 新建环境
4. 进入 `静态网站托管`
5. 开通静态网站托管
6. 上传这些文件：
   - `index.html`
   - `styles.css`
   - `app.js`
   - `questions-data.js`
7. 默认首页设置为 `index.html`
8. 复制系统分配的访问地址测试
9. 如果你有自己的域名，再绑定域名并开启 HTTPS

说明：如果使用自己的国内域名，通常需要先完成 ICP 备案。没有域名也可以先用平台分配的默认访问地址。

## 长期备选方案：腾讯云 COS 静态网站

适合只托管静态文件，成本低。

操作步骤：

1. 打开腾讯云控制台
2. 搜索 `对象存储 COS`
3. 新建存储桶
4. 上传这些文件：
   - `index.html`
   - `styles.css`
   - `app.js`
   - `questions-data.js`
5. 开启静态网站功能
6. 索引文档设置为 `index.html`
7. 权限按公开网站配置访问
8. 如需正式域名，绑定自定义域名并配置 HTTPS

## 快速测试方案：Netlify Drop

适合最快上线。

1. 打开 Netlify Drop：`https://app.netlify.com/drop`
2. 把 `tj-computer-practice-site.zip` 解压
3. 将解压后的文件夹拖进去
4. Netlify 会自动生成一个可访问网址

优点：最快，不需要写代码。  
缺点：国内访问速度不一定稳定。

## 海外长期方案：Vercel

适合以后持续更新。

1. 注册或登录 Vercel
2. 新建项目
3. 上传当前文件夹或连接 GitHub 仓库
4. Framework 选择 `Other`
5. Build Command 留空
6. Output Directory 留空或填 `.`
7. 部署

优点：更新方便。  
缺点：国内访问速度不一定稳定。

## 推荐上线顺序

如果你要长期使用，我建议：

1. 先用 Netlify Drop 测试页面功能
2. 正式使用时部署到腾讯云 CloudBase 静态托管
3. 如果以后要手机电脑同步错题，再接入 CloudBase 数据库和登录

## 后续如果要做账号同步

现在的数据存在浏览器本地，适合自己使用。如果以后要做到：

- 手机、电脑同步错题
- 登录账号
- 多人使用
- 后台更新题库
- 每天自动推送总结

那就需要加后端和数据库。可以用 Supabase、Firebase、腾讯云开发或自己服务器。
